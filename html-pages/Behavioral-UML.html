Activity Diagram

The Activity Diagram is used for describing how and where your application moves and the users who have access to specific information throughout the process, normally separated into swim lane, i will link the video i watched for more specifics ;)

Sequence Diagram

This kind of diagram is also known as an event diagram because it follows Interactions among classes, normally following the messages that are sent through and around, an example is an eshop, you add things to carts and get a total depending on how many items exist. i will link my source here ;)

use case diagram

The use Case Diagram describes how a user would interact with the application and how you would develope to adapt to said class to make suer it works properly afterwords

state diagram

The State Diagram is meant to describe how the application would react if a user were to interact with it, especially useful when you need to describe any effects that need to be had by any features such as hover or focus events, i will link my material :)

Communication diagram

The Communication Diagram is meant to indicate the sequence of how the objects interact, it also is meant to describe both the static and dynamic behaviors of an application

Interaction overview diagram

This kind of diagram is a good compilation of both sequence and activity diagrams, it neatly adds the functionalities of both to make a diagram that makes it so you can see the exact positions of the items and who has access as well as what parts exactly need to be built

Timing diagram

Timing Diagrams are similar to Sequence Diagrams in that theyre both for specific instances of time, except this one is a little more specific and is read from left to right intead of top down
